## 1. 문서 목적

본 문서는 TripSync MVP의 기능 정확성, 일정 생성 품질, 인증/세션 안정성, 모바일 웹 사용성을 검증하기 위한 테스트 범위와 통과 기준을 정의한다.

## 2. 테스트 목표

### 2.1 기능 목표

- 방장 Google OAuth/일반 로그인 성공
- 동행자 비회원 참여 성공
- TPTI 검사/결과 산출 성공
- 갈등 지도 생성 성공
- 합의 일정 3가지 옵션 생성 성공
- 그룹의 옵션 선택 및 일정 확정 성공
- 일정 재생성 및 버전 관리 성공

### 2.2 비기능 목표

- TPTI 결과 응답 1초 이내
- 일정 생성 5초 이내
- 50명 동시 접속에서 치명적 오류 없음
- 모바일 인앱 브라우저 기준 주요 플로우 정상 작동

## 3. 테스트 범위

### 3.1 포함 범위

- 인증(Local/OAuth/Guest)
- TPTI 점수 계산
- 방 생성/참여
- 갈등 지도 계산
- 합의 엔진
- TourAPI 캐시 데이터 기반 일정 생성

### 3.2 제외 범위

- 실결제/예약 연동

## 8. E2E 시나리오

### E2E-001 기본 3인 합의 일정 생성

1. 방장 구글 OAuth 로그인 또는 일반 로그인
2. 방 생성
3. 링크 공유
4. 게스트 2명 링크 접속
5. 각자 TPTI 검사 완료
6. 갈등 지도 확인
7. 방장이 일정 생성
8. 3가지 옵션(균형형/개성형/지역 발굴형) 표시 확인
9. 각 옵션별 동행자별 만족도 표시 확인
10. 그룹이 하나의 옵션 선택 및 확정
11. 확정 일정/슬롯 이유 표시 확인

통과 기준:
- 3화면 흐름이 중단 없이 완료
- 3가지 옵션 모두 표시
- 각 옵션에 동행자별 만족도 표기
- 옵션 선택 후 확정 일정으로 전환
- 각 슬롯에 대상 축/대상 사용자 표기

### E2E-002 강한 충돌 2인 그룹

1. 한 명은 mobility 90, 다른 한 명은 mobility 10
2. 일정 생성
3. 오전/오후 슬롯이 서로 다른 취향을 반영하는지 확인

통과 기준:
- 평균형 장소만 나열되지 않음
- 최소 2개 개인 슬롯 존재

### E2E-003 신규 멤버 참여 후 재생성

1. 2인으로 일정 생성 완료
2. 신규 게스트 참여 및 검사 완료
3. 기존 일정에 재생성 필요 상태 표시 확인
4. 방장이 재생성 실행

통과 기준:
- 기존 일정 version 유지
- 새 version 생성

### E2E-004 OAuth 실패 복구

1. 잘못된 state로 callback 진입
2. 로그인 실패 페이지 확인
3. 재시도로 정상 로그인

통과 기준:
- 잘못된 callback에서 세션 발급되지 않음
- 사용자 친화적 오류 메시지 제공

### E2E-005 LLM fallback 시나리오

1. LLM 응답을 invalid JSON으로 모킹
2. 일정 생성 요청
3. fallback 일정 반환 여부 확인

통과 기준:
- API 500이 아닌 정상 일정 반환
- fallback 사용 로그 기록

### E2E-006 공개 공유 페이지 검증

1. TPTI 결과 공유 URL 접속
2. `/share/tpti/[resultId]` 페이지 렌더링 확인
3. 일정 공유 URL 접속
4. `/share/schedule/[scheduleId]` 페이지 렌더링 확인

통과 기준:
- 비로그인 상태에서도 공유 페이지 접근 가능
- 개인정보 없이 공유용 최소 정보만 노출
- OG 메타 정보가 정상 구성됨

## 9. 수동 QA 체크리스트

### 9.1 모바일 웹 UX

- 모바일 인앱 브라우저에서 로그인/링크 진입 정상 동작
- TPTI 문항이 한 화면에 과도하게 길지 않음
- 진행률 바가 올바르게 증가함
- 결과 카드/갈등 지도/일정 타임라인이 모바일 너비에서 깨지지 않음

### 9.2 공유 링크

- 잘못된 공유 코드 오류 처리
- 이미 참여한 사용자의 재진입 처리
- 공유 링크 미리보기(OG) 노출 확인
- `/share/tpti/[resultId]` 공개 페이지 렌더링 확인
- `/share/schedule/[scheduleId]` 공개 페이지 렌더링 확인
- soft delete된 공유 대상은 비공개/404 처리 확인

### 9.3 일정 화면

- 3가지 옵션(균형형/개성형/지역 발굴형)이 모두 표시됨
- 각 옵션에 동행자별 만족도(%)가 표시됨
- 옵션 선택 후 확정 일정으로 전환됨
- 확정 일정의 시간 순서가 올바름
- 중복 장소가 과도하지 않음
- 각 슬롯의 reason label이 읽기 쉬움
- 인구감소지역 장소에 "숨은 명소" 배지 표시 확인
