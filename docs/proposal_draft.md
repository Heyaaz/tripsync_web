# TripSync 서비스 제안서

---

## 1) 서비스 기획배경 및 필요성

### 1. 서비스 기획 배경

친구들과 여행 계획을 짜다 보면 늘 비슷한 결말이었다. 의견이 오가다 결국 모두가 아는 곳으로 수렴됐다. 처음에는 취향 차이 때문이라고 생각했다. 하지만 반복할수록 더 근본적인 원인이 보였다. 아무도 낯선 장소를 끝까지 주장하지 않았다. 의견 충돌이 두려웠기 때문이다. "별로면 어쩌지"라는 부담, 일정이 실패했을 때 책임을 져야 한다는 압박이 모두를 검증된 유명 관광지로 이끌었다.

구조적인 문제였다. 그룹 여행에서 취향 충돌이 해결되지 않는 한, 여행자는 안전한 선택만 반복한다. 일정 제안자는 "독박"을 피하기 위해 모두가 아는 곳을 고른다.

TripSync는 이 경험에서 출발했다. 여행의 문제는 정보 부족이 아니라 합의의 어려움이다.

---

### 2. 서비스 필요성

**그룹 여행의 갈등 구조는 지방 소외를 심화시킨다.**

현재 여행 플래닝 서비스(트리플, 마이리얼트립 등)는 모두 **개인 맞춤 추천**에 집중되어 있다. 그룹에서 다시 의견이 충돌하는 문제는 해결하지 못한다. 카카오톡 투표는 단순 찬반만 가능하고, 스프레드시트 공유는 방장 혼자의 수고로 귀결된다. 그룹 합의를 구조적으로 만들어주는 서비스가 없다.

취향 충돌이 해결되지 않는 한, 어느 지역을 가든 여행자는 안전한 선택만 반복한다. 알려진 곳으로만 수렴되고, 인구감소지역의 숨은 명소는 닿지 못한다.

**갈등 없는 여행이 지방을 살린다.**

그룹 취향 충돌이 해결되면, 비로소 유명 관광지 너머까지 탐색할 여유가 생긴다. 인구감소지역의 숨은 명소는 강요나 캠페인 없이도 자연스럽게 일정에 포함될 수 있다. TripSync는 그 조건을 만드는 서비스다.

## 2) 서비스 개요

### 1. 기획 서비스 소개 (한 줄)

> **"동행자 취향 충돌을 데이터로 조율하고, 그룹이 선택할 수 있는 충남 여행 일정 선택지를 제시하는 그룹 여행 의사결정 플랫폼"**

---

### 2. 기획 서비스 주요 기능

TripSync의 핵심 흐름은 **취향 수집 → 갈등 시각화 → 복수 일정 생성 → 투명한 선택**으로 이어진다.

#### 기능 1. TPTI 검사 (Trip Personal Type Indicator)

여행 취향을 4개 축, 8문항으로 정량화하는 도구다. 30초 이내 완료를 목표로 설계했다.

| 축 | 설명 | 양극단 |
|----|------|--------|
| 활동성 (M) | 얼마나 많이 움직이며 탐험할 것인가 | 뚜벅이 체험형 ↔ 휴식 호캉스형 |
| 기록 (P) | 사진과 SNS에 어느 정도 비중을 두는가 | 인생샷 SNS형 ↔ 눈으로 담는 실속형 |
| 예산 (B) | 지출에 대한 감각은 어느 쪽인가 | 가심비 플렉스형 ↔ 가성비 절약형 |
| 테마 (T) | 어떤 분위기의 여행지를 선호하는가 | 자연 힐링형 ↔ 도심 핫플형 |

TPTI는 심리 분류 도구가 아니라 **동행자 간 상대적 차이를 비교 가능하게 만드는 대화 구조화 도구**다. 정확한 유형 분류보다 취향의 차이를 눈에 보이게 만드는 것이 목적이다.

#### 기능 2. 그룹 갈등 지도 (Conflict Map)

그룹 내 모든 동행자의 TPTI 결과를 한 화면에 겹쳐 보여준다. 어느 축에서 취향이 충돌하는지, 어느 축에서 공통 지대가 형성되는지를 레이더 차트로 시각화한다.

- **공통 지대**: 모든 동행자 점수가 유사한 축 → 일정의 기본 배경으로 확정
- **충돌 축**: 점수 편차가 큰 축 → 시간 배분(Time Slicing)으로 교대 반영

충돌 지도는 "내가 틀린 게 아니라 우리가 다른 것"임을 데이터로 보여줌으로써 대화의 전제를 바꾼다.

#### 기능 3. 복수 일정 선택지 생성

시스템이 단일 일정을 강요하지 않는다. 서로 다른 특성의 **3가지 일정 선택지**를 생성하여 그룹이 선택하도록 한다.

| 옵션 | 특성 |
|------|------|
| **균형형** | 공통 지대 중심. 모든 동행자의 점수를 균등하게 반영한 안전한 선택 |
| **개성형** | 충돌 축의 양쪽 취향을 시간 배분으로 교대 반영. 절충안 |
| **지역 발굴형** | 충남 인구감소지역 숨은 명소를 최소 1곳 이상 포함. 새로운 탐험 중심 |

#### 기능 4. 개인별 만족도 투명화

각 일정 옵션에 동행자별 예상 만족도를 함께 표시한다.

> 예시: `균형형 일정 — A님 72% / B님 68% / C님 75%`

수치를 투명하게 공개함으로써 "내가 일정 짠 게 아니라 데이터가 합의한 것"이라는 구조를 만든다. 방장의 책임 부담이 시스템으로 분산된다.

#### 기능 5. 숨은 명소 배지 + 기여 카드

인구감소지역 숨은 명소를 방문한 이후 배지를 발급하고, 해당 장소를 일정에 포함하게 된 경위를 요약한 기여 카드를 제공한다. 여행 후 SNS 공유를 유도하여 2차 확산을 도모한다.

### 3. 서비스 차별성

| 구분 | 기존 서비스 | TripSync |
|------|------------|---------|
| **문제 인식** | 개인 추천 부족 | 그룹 합의 구조 부재 |
| **해결 방식** | 개인 맞춤 추천 제공 | 취향 충돌 측정 → 복수 선택지로 협상 테이블 제공 |
| **책임 구조** | 방장 개인이 선택·책임 | 시스템이 선택지 생성, 그룹이 함께 선택 |
| **지방 연계** | 없음 (유명 관광지 편중) | 인구감소지역 숨은 명소를 강요 없이 자연스럽게 포함 |
| **결과물** | 단일 추천 일정 | 서로 다른 특성의 3가지 일정 선택지 |

기존 서비스들은 "어디를 갈지 몰라서" 추천을 받는 개인을 타겟으로 한다. TripSync는 "어디를 갈지 정하지 못하는" 그룹의 합의 문제를 타겟으로 한다. 문제의 정의 자체가 다르다.

핵심 차별점은 **취향 충돌을 데이터로 가시화하고, 복수의 선택지로 협상 구조를 만든다**는 것이다. 시스템이 결정을 대신하는 것이 아니라, 그룹이 선택할 수 있는 환경을 설계한다.

### 4. 서비스 내 지역 특화 관련 사항 (충남 특화)

#### MVP 대상 지역: 충청남도 4개 권역

충남은 해양·자연·역사·문화 등 다양한 관광 자원을 보유하고 있음에도, 실제 방문은 일부 유명 관광지에 집중되는 구조가 뚜렷하다. 그룹 여행에서 선택되지 못하는 장소들이 존재하는 지역으로, 서비스 효과를 검증하기에 가장 적합한 대표 사례다.

| 권역 | 주요 앵커 장소 | 연계 인구감소지역 숨은 명소 (예시) |
|------|------------|-------------------------------|
| 공주·부여권 | 공산성, 부소산성 | 마곡사 주변 숲길, 외산면 무량사 |
| 태안·보령권 | 꽃지해수욕장, 대천해수욕장 | 천리포수목원, 보령 성주사지 |
| 천안·아산권 | 독립기념관, 현충사 | 광덕산 자연휴양림, 아산 외암민속마을 |
| 서산·당진권 | 해미읍성, 간월도 | 서산 버드랜드, 당진 삼선산수목원 |

#### 인구감소지역 연계 전략

행정안전부 지정 인구감소지역(충남 내 해당 시·군)의 관광지에 가중치를 적용한다. 강요나 캠페인 방식이 아니라 **취향에 맞는 장소가 자연스럽게 일정에 포함되는 구조**로 설계한다.

- **앵커-숨은명소 동선 연계**: 유명 앵커 장소와 이동 경로상 인접한 인구감소지역 숨은 명소를 묶어 하나의 코스로 구성
- **지역 발굴형 옵션 필수 포함**: 3가지 일정 선택지 중 "지역 발굴형"에는 충남 인구감소지역 숨은 명소를 반드시 1곳 이상 포함
- **가중치 부여 로직**: 인구감소지역 시·군 코드 기반으로 TourAPI 관광지 데이터에 가산점을 적용하여 일정 후보 생성 단계에서 자연스럽게 포함

#### 충남 우선 전략의 근거

외갓댁이 충청남도 예산에 있고, 대학을 천안에서 다녔다. 충남은 오랜 기간 친숙하게 접해온 지역이다. 관광 자원은 다양한데 실제 방문은 유명 관광지에만 집중되는 현상을 반복적으로 목격했다. TripSync는 충남을 시작점으로, 그룹 여행의 합의 구조가 지방 소외를 어떻게 바꿀 수 있는지를 증명하는 대표 사례로 만들고자 한다. 충남에서 검증된 구조를 이후 유사한 문제를 가진 다른 지방으로 확장한다.

## 3) 데이터 활용 방안

### 1. 활용 예정 한국관광공사 OpenAPI

| API | 엔드포인트 | 활용 목적 |
|-----|-----------|---------|
| **국문 관광정보 서비스 (TourAPI 4.0)** | `areaBasedList`, `detailCommon` | 지역별 관광지 목록 및 상세정보 수집 |
| **위치 기반 관광정보 조회** | `locationBasedList` | 관광지 좌표 정보, 동선 현실성 보정 |
| **행사·공연·축제 정보 조회** | `searchFestival` | 행사 포함형 일정 옵션 구성 |
| **지역 기반 관광지 상세** | `detailIntro`, `detailInfo` | 운영시간, 입장료 등 세부 정보 |

---

### 2. 데이터 활용 방식

#### (1) 관광지 데이터 → 일정 후보 생성

TourAPI의 `areaBasedList`를 통해 충남 4개 권역의 관광지 전체 목록을 수집한다. 수집된 데이터는 콘텐츠 타입(관광지, 문화시설, 레포츠 등)과 지역 코드로 분류하고, 인구감소지역 시·군 코드에 해당하는 장소에 가중치를 부여한다. 이 가중치가 "지역 발굴형" 옵션에서 숨은 명소가 자연스럽게 선택되도록 작동하는 핵심 로직이다.

```
TourAPI areaBasedList (충남 지역 코드)
       ↓
콘텐츠 타입별 분류
       ↓
인구감소지역 시·군 코드 매핑 → 가중치 부여
       ↓
TPTI 벡터와 결합하여 일정 후보 풀 생성
```

#### (2) 좌표 정보 → 동선 현실성 보정

관광지별 위도·경도 데이터를 활용하여 직선거리 기반으로 관광지를 정렬한다. 정확한 경로 최적화가 목적이 아니라, 비현실적인 이동 경로(예: 태안에서 천안을 거쳐 다시 공주로 가는 경로)를 제거하는 것이 목적이다. 동선은 서비스의 핵심 기능이 아닌, 일정의 현실성을 확보하기 위한 보조 기능으로 정의한다.

#### (3) 행사·축제 데이터 → 선택지 다양화

`searchFestival` API로 여행 날짜에 맞는 충남 지역 행사 및 축제 데이터를 수집한다. 해당 데이터는 3가지 일정 선택지 중 별도 옵션으로 구성하거나, 기존 옵션에 행사 방문 슬롯을 추가하는 방식으로 활용한다. 행사는 일정의 중심이 아니라, 옵션을 풍부하게 만드는 요소로만 사용한다.

#### (4) 인구감소지역 코드 필터링 → 숨은 명소 가중치

행정안전부 지정 인구감소지역 시·군 코드를 TourAPI 지역 코드와 매핑하여, 해당 지역 관광지에 일정 생성 단계에서 가산점을 적용한다. 이 방식은 데이터 기반으로 충남 인구감소지역 방문을 자연스럽게 유도하는 핵심 메커니즘이다.

## 4) 서비스 발전 방향

### 1. 개발 서비스 향후 발전방향

#### Phase 1: MVP 검증 (충남 집중)

현재 단계. 충남 4개 권역을 대상으로 TPTI 검사, 갈등 지도, 복수 일정 생성의 핵심 흐름을 검증한다. 실제 그룹 여행에서 합의 구조가 작동하는지, 인구감소지역 숨은 명소가 자연스럽게 선택되는지를 사용자 데이터로 확인한다.

#### Phase 2: 지방 전국 확장

충남에서 검증된 구조를 수도권을 제외한 지방 전역으로 확장한다.

| 확장 대상 | 특성 |
|---------|------|
| 강원도 | 자연·액티비티 특화. 인구감소지역 비율 높음 |
| 전라남도 | 문화·역사·해안. 관광 자원 다양성 큼 |
| 경상북도 | 역사 유적지 중심. 방문 편중 심함 |
| 제주도 | 고밀도 관광 지역. 오버투어리즘 분산 효과 기대 |

#### Phase 3: 지자체 B2B 리포트

인구감소지역 방문 데이터를 익명으로 집계하여 지자체에 제공하는 B2B 서비스를 구축한다.

- **데이터 내용**: 방문 경로, 체류 시간대, 동행 그룹 규모, TPTI 취향 분포
- **활용 목적**: 관광 콘텐츠 개발, 인프라 투자 우선순위 결정, 시즌별 행사 기획
- **수익 모델**: 지자체 관광 데이터 구독 서비스

지자체 입장에서 TripSync는 단순한 여행 앱이 아니라, 실제 방문자 데이터 기반의 관광 정책 보조 도구가 된다.

#### Phase 4: 합의 알고리즘 고도화

여행 후 수집되는 만족도 피드백을 학습 데이터로 활용하여 합의 엔진의 정확도를 높인다.

- **피드백 루프**: 여행 후 동행자별 실제 만족도 수집 → TPTI-만족도 상관관계 분석
- **가중치 개인화**: 반복 사용자에게는 과거 만족도 패턴 기반 취향 보정 적용
- **지역 데이터 갱신**: 인구감소지역 장소별 방문 후 평가를 누적하여 숨은 명소 품질 관리

> 취향 충돌이 해결되면 그룹은 유명 관광지 너머를 탐색하게 되고,
> 그 결과로 충남의 인구감소지역에 사람이 자연스럽게 흐른다.
